Execution rules:

- Update CUE contracts and fixtures before their Go or procedural consumers.
- Bind candidate, fixture, probe, and evaluator contracts to the complete
  migrated kernel through CUE package imports or same-package references. Do
  not reproduce kernel-shaped schemas or reimplement kernel proofs in Go.
- Generate probes from kernel operations and candidate declarations; do not
  accept arbitrary expected-state harnesses.
- Keep valid-state conflict fixtures separate from schema-invalid ingress.
- Require `cueprobe` to derive the effective ProbeSubject from actual loaded
  sources, module identity, normalized build options, selector inputs, runner
  protocol, and CUE engine version. Treat `subjectExpectation` only as an
  expectation and fail on mismatch.
- Compare pre-load and post-operation digests for the module manifest, declared
  sources, selector inputs, and build inputs. Emit `source-changed` and suppress
  semantic facts on mutation.
- Treat structured ProbeSubject equality as authoritative. Project exact CUE
  numbers to the bounded `#CanonicalSubjectValue` tagged coefficient/exponent
  representation before RFC 8785 serialization and SHA-256 projection.
- Emit semantic bottom only from the structured CUE Go API channel after load,
  lookup, and operand preconditions succeed.
- Record CLI formatting and vet failures only as structural-gate observations.
- Keep timeout, protocol, syntax, selector, package, and infrastructure failure
  distinct from semantic bottom.
- Keep the Go runner free of verdict, satisfaction, coverage, and admission
  policy.
- Reject unknown fields and claimant-provided conclusions at every external
  ingress.
- Produce bounded LSP observations through `cueprobe lsp-observe`; keep them
  advisory and separately exportable.
- Export rejected, incomplete, and runner-failure bundles without hiding their
  state.
- Do not allow generated observations, reports, or bundles to become authority.
- Define the closed skill-interface manifest before updating `SKILL.md`.
  Validate its bounded frontmatter reference and exact identifiers through
  `cueprobe skill-check`; do not parse prose as architecture.

kernel package and its consuming packages, and keep structural gates separate
from semantic probes. At minimum run the applicable forms of:

```bash

cue vet -c=false <declared-package>
cue vet -c <declared-concrete-surface>
```

For the runner:


Run only the closed declared command surfaces:

```bash

cueprobe lsp-observe --request <request> --output <lsp-observation>
cueprobe skill-check --skill <SKILL.md> --manifest <interface-manifest>
```

Validate raw output against `#ProbeObservation` or `#LSPObservation`, replay
probe facts through CUE evaluation, validate interface references through CUE,
and run the required positive, negative, invariant, compatibility, and
adversarial scenarios.

Completion for each unit requires:

- its selected requirements include a verified complete dependency closure;
- every cited acceptance criterion has complete scenario coverage;
- package metadata, probe specs, subjects, requests, observations, and artifact
  ingress are closed;
- valid conflict fixtures and schema-invalid ingress fixtures prove their
  distinct boundaries;
- runner stage facts distinguish semantic bottom from every execution failure;
- the runner-derived subject matches any supplied expectation and pre/post
  source digests remain stable;
- exact numbers round-trip through the bounded canonical subject value;
- bounded LSP and skill-interface observations validate without becoming
  semantic admission;
- structural gates and semantic evaluation both pass independently;
- accepted and rejected candidates behave as declared;
- artifact bundles are deterministic, complete, and single-subject;
- CUE computes unit satisfaction and the exported value is literally `true`;
- no P1 adapter or workbook semantics enter the P0 implementation.

Do not treat Go, CLI, LSP, or shell success as admission. CUE-derived
satisfaction is the terminal gate.
