Node objectives:

snapshot.verify
    Resolve #103 and #104 from verified user-supplied issue transports when
    available, then verify the pinned #103 transport and normalized bootstrap
    snapshot. Reacquire only a missing or unverifiable issue payload.

workbook-identity.admit
    Encode and validate the issue-selected bootstrap workbook identity.

project.create
    Replace the empty pyproject.toml placeholder with the canonical Factory
    uv project.

lock.create
    Generate uv.lock from the project and prove `uv lock --check` succeeds.

contracts.create
    Define the closed bootstrap, workflow, scenario, evidence, command, and
    output contracts under marimo/workflows/bdd/.kb.

workbook.create
    Implement only Python-assigned workflow nodes in
    marimo/workflows/bdd/validate_implementation_unit.py. Do not add PEP 723
    metadata or admission logic.

fixtures.create
    Implement all positive, negative, invariant, compatibility, and
    adversarial scenarios declared by issue #104.

bootstrap.execute
    Run locked and exact validation in disposable runtime state and admit the
    resulting transient evidence through CUE.

self-conformance.admit
    Execute the canonical suite against this implementation unit and require
    the CUE-exported self-conformance admission value to be literally true.

provisional.retire
    Prove provisional admission is retired and unavailable to later or
    repeated implementation units. This is the terminal node.
