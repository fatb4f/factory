Implement the revised P0 contract from:

    https://github.com/fatb4f/factory/issues/106

The target is the executable kernel–probe–eval CUE skill package on branch
`feat/cue-skill`, including the single admitted Go `cueprobe` runner. This file
is execution guidance only; it does not replace issue #106 or vetted CUE
contracts.

Implementation objective:

```text
skill-local executable kernel
→ versioned package and closed ProbeSpec
→ closed structured ProbeSubject
→ closed machine-readable skill-interface manifest
→ source-controlled Go cueprobe subject and observation adapter
→ closed raw ProbeObservation
→ CUE-derived probe/family/candidate/coverage/suite evaluation
→ distinct structural CLI gates and advisory LSP observations
→ deterministic complete semantic artifact bundle
→ statically constrained thin SKILL.md
```

Implementation-unit procedure:

1. Select the P0 requirement IDs to satisfy in the current unit.
2. Compute their complete transitive closure from the issue's direct
   `depends_on` edges.
3. Record every requirement and acceptance-criterion ID in scope.
4. Declare a validation DAG preserving the dependency order.
5. Map every cited acceptance criterion to all required scenario classes.
6. Identify authoritative inputs, implementation artifacts, commands,
   fixtures, raw observations, CUE evaluations, and exported evidence.
7. Execute only nodes whose prerequisites have validated evidence.

Keep implementation within:

```text
.codex/skills/cue/SKILL.md
.codex/skills/cue/agents/openai.yaml
.codex/skills/cue/interface/**
.codex/skills/cue/contracts/**
.codex/skills/cue/runner/go.mod
.codex/skills/cue/runner/go.sum
.codex/skills/cue/runner/cmd/cueprobe/**
```

Validation must use manifest-declared coordinates, validate both the migrated
kernel package and its consuming packages, and keep structural gates separate
from semantic probes. At minimum run the applicable forms of:

```bash
cue fmt --check --files <declared-files>
cue vet -c=false <declared-package>
cue vet -c <declared-concrete-surface>
```

For the runner:

```bash
cd .codex/skills/cue/runner
go mod verify
go test -mod=readonly ./...
go build -mod=readonly ./cmd/cueprobe
```

Run only the closed declared command surfaces:

```bash
cueprobe observe --request <request> --output <probe-observation>
cueprobe lsp-observe --request <request> --output <lsp-observation>
cueprobe skill-check --skill <SKILL.md> --manifest <interface-manifest>
```

Validate raw output against `#ProbeObservation` or `#LSPObservation`, replay
probe facts through CUE evaluation, validate interface references through CUE,
and run the required positive, negative, invariant, compatibility, and
adversarial scenarios.

Completion for each unit requires:

- its selected requirements include a verified complete dependency closure;
- every cited acceptance criterion has complete scenario coverage;
- package metadata, probe specs, subjects, requests, observations, and artifact
  ingress are closed;
- valid conflict fixtures and schema-invalid ingress fixtures prove their
  distinct boundaries;
- runner stage facts distinguish semantic bottom from every execution failure;
- the runner-derived subject matches any supplied expectation and pre/post
  source digests remain stable;
- exact numbers round-trip through the bounded canonical subject value;
- bounded LSP and skill-interface observations validate without becoming
  semantic admission;
- structural gates and semantic evaluation both pass independently;
- accepted and rejected candidates behave as declared;
- artifact bundles are deterministic, complete, and single-subject;
- CUE computes unit satisfaction and the exported value is literally `true`;
- no P1 adapter or workbook semantics enter the P0 implementation.

Do not treat Go, CLI, LSP, or shell success as admission. CUE-derived
satisfaction is the terminal gate.
